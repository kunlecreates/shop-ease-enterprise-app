## Testing Approach

All tests follow these patterns:
- Use validateStatus: () => true to handle all HTTP status codes
- Test both success and failure scenarios
- Use timestamp-based unique identifiers to prevent collisions
- Validate response structure with JSON schemas where applicable
- Support conditional assertions (if 200, then validate data)
- Tests are environment-aware (can run in ci, local, or staging)
